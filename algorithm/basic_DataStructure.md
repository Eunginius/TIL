# 배열이란?
같은 자료형의 변수로 이루어진 요소가 모여 직선으로 줄지어 있는 자료구조
  
  
### 선언
자료형 배열_이름[요소_개수];   
### 초기화
선언 = {초기화 할 값들을 comma(,)로 구분}   
중괄호 안의 값들이 0부터 (요소_개수 - 1) 인덱스까지 순서대로 배열에 저장된다.   

### 메모리 할당과 동적 객체 생성

#### calloc 함수 
- 헤더: #include <stdlib.h>   
- 형식: void *calloc(size_t nmemb, size_t size);   
- 해설: 크기가 size인 자료가 nmemb개만큼 들어갈 메모리를 할당한다. 할당 후 모든 비트가 0으로 초기화횐다.   
- 반환: 메모리 할당에 성공 -> 할당 영역 첫 번째 포인트, 실패 -> NULL 포인트   
   
#### malloc 함수
- 헤더: #include <stdlib.h>   
- 형식: void *malloc(size_t size);   
- 해설: 크기가 size인 메모리를 할당한다. 할당된 메모리 값은 정의되지 않는다. 
- 반환: 메모리 할당에 성공 -> 할당 영역 첫 번째 포인트, 실패 -> NULL 포인트   

#### free 함수
- 헤더: #include <stdlib.h>   
- 형식: void free(void *ptr); 
- 해설: ptr이 가리키는 메모리를 해제하여 다시 할당할 수 있도록 한다.    
        프로그램 실행 도중에도 원하는 시점에 변수 생성 또는 제거 가능해진다.   
        NULL 포인터인 경우이거나,      
        calloc, malloc, realloc 함수에 의해 반환된 포인터가 아니거나,   
        이미 해제된 영역이면 아무것도 하지 않는다.
- 반환: 메모리 할당에 성공 -> 할당 영역 첫 번째 포인트, 실패 -> NULL 포인트   

### C 언어의 메모리 구조
프로그램 실행 -> OS가 프로그램이 사용할 메모리 영역 할당   
메모리 영역: Data, Stack, Heap   

#### Data
- 전역 변수와 static 변수가 할당되는 공간
- 프로그램 시작 시 할당, 종료 시 해제

#### Stack
- 함수 호출 시 생성되는 지역 변수와 매개변수 저장되는 공간
- 함수 호출 완료 시 사라짐

#### Heap
- 필요에 따라 동적으로 메모리 할당
- 할당해야 할 메모리 영역 크기를 run time에 결정해야 하는 경우 사용 (동적이니까 어느 시점에서 얼만큼 할당할지 모르기 때문)

### 포인터
#### 배열 요소 접근 식
a[i], *(a+i)   

#### 배열 요소 주소 식
&a[i], a+i   

## 배열 요소의 최댓값 구하기
1. 첫 번째 요소를 max 변수에 대입
2. 조건문 실행
3. 그 다음 요소가 max보다 크면 max 값을 갱신
4. 마지막 요소까지 반복 -> *tranverse 

##### *traverse: 주사, 배열의 요소를 하나씩 차례로 살펴보는 과정

## 배열 요소를 역순으로 정렬하기   
- 반복문 선언
   - 첫 번째 요소와 마지막 요소를 swap
   -  두 번째 요소와 마지막-1 요소 swap 하는 방식으로 반복 (배열 길이/2 만큼 반복한다. 그럼 홀수인 경우는 정중앙 값만 그대로 유지)

## 기수 변환하기
1. 0~Z까지 저장해 둘 배열 dchar과 변환한 진수를 저장할 배열 d를 선언   
2. 변환할 진수 n과 정수 x를 입력 받기   
3. dchar[x%n] 값을 d에 저장 후 d 인덱스를 증가(그 다음 자릿수를 저장하기 위함)   
4. x/=n   
-> 3, 4 반복 (x가 0보다 큰 동안 반복 // 4번 절차에 의해 언젠간 0 이하가 된다.)   
5. 배열 d를 역순으로 정렬 (0부터 배열 끝까지 거꾸로 채워 넣었기 때문)
